## 2. 프로젝트 개요

- 솔루션 한 줄 요약 : Hyperliquid DEX의 실시간 데이터를 Rule Engine으로 정량 분석하고, LLM이 한국어로 해석·전략·코인 추천까지 제공하는 AI Copilot 트레이딩 터미널.

## 4. AI 활용 방식 및 주요 프롬프트

### 4.1 개발 과정에서의 AI 활용

프로젝트의 전 과정을 AI와 대화하며 진행했다. 아이디어 도출부터 요구사항 정의, 와이어프레임, 설계, 구현, 브랜딩, 문서화까지 모든 단계에서 AI가 산출물을 생성하고, 사람이 검토·피드백하는 사이클을 반복했다.

| 단계 | AI 도구 | 산출물 |
|------|---------|--------|
| 아이디어 도출 | ChatGPT | 프로젝트 컨셉 확정 |
| 요구사항 정의 | Kiro | requirements.md (인수 조건 포함) |
| 와이어프레임 | Kiro | wireframe.html 3종 |
| 기술 설계 | Kiro | design.md, api-spec.md |
| 구현 (FE+BE ~30파일) | Kiro | 전체 소스 코드 |
| 브랜딩 (로고/아이콘) | Gemini | 로고, 컬러 팔레트 |
| 문서화 | Kiro | README 3종 + 상세 문서 3종 |

### 4.2 서비스 내 AI 활용 (런타임)

서비스 자체가 LLM을 활용하여 트레이더에게 실시간 분석을 제공한다. Rule Engine이 산출한 수치 데이터를 LLM에게 전달하면, LLM이 한국어로 해석·요약·전략을 생성한다.

- LLM 멀티 프로바이더 폴백 : Claude Sonnet 4 → Groq (Llama 3.3 70B) → Gemini 2.0 Flash → OpenAI (GPT-4o-mini)
- Rule Engine + LLM 역할 분리 : LLM이 실패해도 Rule Engine 수치 결과는 독립적으로 표시

### 4.3 기존 방식 대비 달라진 점

- 기존 : 트레이더가 펀딩 레이트, OI, 청산 데이터 등을 각각 확인하고 머릿속에서 종합 판단
- 변경 : Rule Engine이 5개 지표를 정량화하고, LLM이 종합 해석 + 구체적 TP/SL + 코인 추천까지 자동 생성

### 4.4 실제 사용한 주요 프롬프트

## 5. 주요 기능 설명

### 기능 1: AI Copilot 포지션 분석

Hyperliquid 지갑 주소를 입력하면 보유 포지션을 실시간 조회하고, 코인별로 5개 지표(리스크 스코어, 지지/저항, 펀딩 레이트, OI, 청산 클러스터)를 Rule Engine이 정량 분석한다. LLM이 각 지표를 한국어로 해석하고, 단기/중기 TP·SL 전략까지 제시한다.

### 기능 2: AI Copilot 오더 분석

미체결 주문을 분석하여 전략 패턴(그리드/레인지/돌파/매집)을 탐지하고, 각 주문의 체결 가능성, 주문 집중 구간, 기존 포지션에 미치는 영향을 평가한다. LLM이 종합 해석과 제안을 생성한다.

### 기능 3: AI 코인 추천 (Discover)

Hyperliquid 전체 마켓 데이터(가격, 거래량, 펀딩 레이트, OI 등)를 수집하여 LLM에게 전달하면, 단기 트레이딩에 적합한 코인 3~5개를 방향(LONG/SHORT), 목표가, 손절가, 신뢰도, 추천 사유와 함께 카드 형태로 제공한다.

## 7. 아쉬운 점 / 개선 여지

- 외부 청산 데이터 API(Coinglass 등)를 연동하지 못해 청산 클러스터 분석이 사용자 포지션 기반으로만 동작한다. 실제 시장 전체의 청산 데이터가 있으면 분석 정확도가 크게 올라갈 것이다.
- LLM 응답 속도가 분석 전체 지연의 주요 원인이다. Claude Sonnet 4 기준 포지션 분석에 LLM 2회 호출(해석 + 전략)이 필요해 10~15초 소요된다. 스트리밍 응답이나 캐싱 전략이 필요하다.
- 무료 LLM API의 한계(쿼터 제한, 모델 폐기)를 경험했다. 프로덕션에서는 유료 플랜이 필수적이다.
- 전략 변경 탐지(주문 수정/취소 이벤트 기반) 기능은 UI만 준비되어 있고 구현이 완료되지 않았다.
- Supercycl 본 서비스에 통합할 때는 멀티 DEX(dYdX, GMX 등) 지원과 사용자 트레이딩 히스토리 기반 개인화 추천이 필요하다.

## 8. 기대 효과

- Supercycl Perp Aggregator 서비스에 AI 추천 기능을 도입하기 전, 프로토타입으로 기술 검증과 사용자 반응을 확인할 수 있다. 실제 트레이더가 AI 분석을 어떻게 활용하는지 데이터를 수집하여 본 서비스 고도화 방향을 결정하는 근거로 활용한다.
- Rule Engine + LLM 분리 아키텍처를 통해 AI 실패 시에도 정량 분석 결과를 독립적으로 제공할 수 있어, 서비스 안정성과 사용자 신뢰를 동시에 확보한다.
- 개발 과정 자체가 AI 기반 개발 프로세스(아이디어 → 요구사항 → 설계 → 구현)의 실증 사례로, 소규모 팀이 AI를 활용해 풀스택 서비스를 빠르게 구축할 수 있음을 보여준다.
